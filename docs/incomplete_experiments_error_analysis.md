## 가능한 중단 원인

### 1. 프로세스 강제 종료 (가능성: 높음)

**증거**:
- 모든 실험이 거의 동시에 중단
- 마지막 iteration 파일은 정상적으로 저장됨 (에러 없음)
- 시스템 재부팅 기록 없음 (마지막 재부팅: 2025-10-15)

**가능한 시나리오**:
- 사용자가 `killall python` 또는 유사한 명령으로 프로세스 종료
- 시스템 관리 스크립트에 의한 종료
- 다른 프로세스와의 충돌

### 2. 메모리 부족 (OOM Killer) (가능성: 중간)

**증거**:
- 여러 실험이 동시에 실행 중이었음 (병렬 실행)
- 각 실험은 약 500MB 메모리 사용 (현재 실행 중인 프로세스 기준)
- 8개 실험 동시 실행 시 약 4GB 메모리 필요

**확인 방법**:
```bash
# 시스템 로그 확인 (권한 필요)
sudo dmesg | grep -i "killed\|oom"
sudo journalctl -k | grep -i "killed\|oom"
```

### 3. 네트워크/API 문제 (가능성: 낮음)

**증거**:
- LLM API 호출 실패 시 재시도 메커니즘이 있음 (`max_retries=3`)
- 마지막 iteration 파일이 정상적으로 저장됨 (API 호출은 성공했을 가능성)

**가능한 시나리오**:
- API rate limit 초과
- 네트워크 연결 일시적 끊김
- API 서버 문제

### 4. 디스크 공간 부족 (가능성: 낮음)

**확인 방법**:
```bash
df -h
```

## 권장 조치사항

### 1. 즉시 조치

1. **현재 실행 중인 프로세스 확인**
   ```bash
   ps aux | grep "python.*main.py"
   ```

2. **시스템 리소스 확인**
   ```bash
   free -h  # 메모리 확인
   df -h    # 디스크 공간 확인
   ```

3. **시스템 로그 확인** (권한 필요)
   ```bash
   sudo dmesg | grep -i "killed\|oom" | tail -20
   sudo journalctl --since "2026-01-20 00:00" --until "2026-01-20 01:00" | grep -i "python\|killed\|oom"
   ```

### 2. 재실행 전 확인사항

1. **메모리 여유 확인**: 8개 실험 동시 실행 시 충분한 메모리 필요
2. **디스크 공간 확인**: 각 실험당 약 수백 MB 로그 생성
3. **API rate limit 확인**: LLM API 호출 제한 확인
4. **네트워크 연결 확인**: 안정적인 인터넷 연결 필요

### 3. 재실행 권장사항

1. **순차 실행 고려**: 메모리 부족 가능성을 줄이기 위해
2. **모니터링 스크립트 추가**: 실험 진행 상황 자동 모니터링
3. **자동 재시작 메커니즘**: 실패 시 자동으로 재시작하는 스크립트
